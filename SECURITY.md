# Security Policy

## Supported Versions

The following versions of **WinHider** are currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of **WinHider** seriously. If you find a vulnerability, please refrain from opening a public issue to prevent potential exploitation while we work on a fix.

### How to Report
Please report security vulnerabilities privately via one of the following methods:

1.  **GitHub Private Reporting (Preferred):**
    * Go to the [Security tab](https://github.com/aamitn/winhider/security) on the repository.
    * Click on "Report a vulnerability" to open a private advisory draft.

2.  **Email:**
    * If private reporting is not available, email us directly at **amitnandileo@gmail.com** OR **amit@bitmutex.com**  with the subject line `[SECURITY] WinHider Vulnerability`.

### What to Include
To help us triage and fix the issue quickly, please include:
* **Description:** A clear description of the vulnerability.
* **Impact:** How the vulnerability can be exploited and what the consequences are.
* **Reproduction Steps:** Step-by-step instructions or a proof-of-concept (PoC) to reproduce the issue.
* **Environment:** The Windows version and WinHider version where the issue was observed.

### Response Timeline
* **Acknowledgment:** We aim to acknowledge your report within **48 hours**.
* **Assessment:** We will assess the severity and impact within **5 business days**.
* **Fix & Disclosure:** If confirmed, we will release a patch as soon as possible. We will coordinate with you on the public disclosure timeline.

Thank you for helping keep WinHider secure!
